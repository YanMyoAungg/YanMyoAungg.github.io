# RAG System Improvement — Design Spec

**Date:** 2026-07-03
**Status:** Approved

## Overview

Upgrade the AI chat's knowledge system from a thin manual profile file to a hybrid build-time extraction + manual overrides approach. The AI roleplays as Yan Myo Aung in first person, auto-discovers facts from the game maps, and offers a resume PDF download when relevant.

## Architecture & Components

```
src/
├── configs/
│   ├── maps/                     ← existing: map configs (unchanged)
│   ├── profile.json              ← NEW: auto-generated by extraction script
│   ├── profile.custom.md         ← RENAMED: was profile.md, now manual overrides
│   └── profile.md                ← deleted
├── utils/
│   └── chatApi.ts                ← MODIFY: reads profile.json + profile.custom.md
├── modules/
│   └── ChatWidget.ts             ← MODIFY: add 📎 download button to chat header
├── scripts/
│   └── extract-profile.ts        ← NEW: build-time extraction script
public/
├── resume.pdf                    ← NEW: user-uploaded CV
└── index.html
```

Changes from current state:
- `profile.md` → deleted, replaced by auto-generated `profile.json` + manual `profile.custom.md`
- `chatApi.ts` → no longer hardcodes guardrails or uses raw markdown; reads both sources
- `ChatWidget.ts` → small addition: resume download button in header
- `outside.ts` → minor fix: degree text corrected from completed to in-progress

## Data Flow

```
npm run build
      │
      ▼
scripts/extract-profile.ts runs
      │  reads src/configs/maps/*.ts
      │  extracts { label, text } from every GameObject with interactions
      │  matches game objects to categories using naming conventions
      │  deduplicates identical entries
      ▼
src/configs/profile.json written
      │
      ▼
ChatApi assembles system prompt at runtime from:
  1. profile.json (auto-extracted facts — skills, experience, education, links, projects)
  2. profile.custom.md (tone, roleplay rules, off-topic handling, resume path)
      │
      ▼
System prompt injected into every API call
```

## Extraction Rules

### What Gets Included

Game objects with interactions are included when they have a `label` property:

| Category | Detection rule | Example |
|---|---|---|
| Identity | Known initial interaction messages (intro text, "My name is...") | "Full-Stack Developer from Myanmar" |
| Skills | Map name contains `techskills` | "JavaScript", "React", "Docker" |
| Experience | Object label matches known company names | "PassionGeek", "G-Next", "Host Myanmar" |
| Education | Object label mentions education keywords | "Computer Science Degree" |
| Links | Interaction contains a URL | GitHub, LinkedIn links |
| Personal | Text reads like a personal statement | "I enjoy building personal projects..." |

### What Gets Filtered Out

- Flavor/game text: "You grab a cold beer", "A wild bug appeared!!", "That's one less for the QA Team!"
- Door/gate messages: "I'm not allowed to enter the building..."
- Furniture interactions: benches, desks without personal content
- Intro welcome message ("Hello! 👋 Welcome to my Portfolio")

### Category Mapping by Naming Convention

The extractor classifies objects using label and map name:

- Map name includes `techskills` → category: **skills**
- Label contains known company names (PassionGeek, G-Next, Host Myanmar, Arkadium, 2C2P) → category: **experience**
- Label or interaction text contains "Degree", "Education", "College", "University" → category: **education**
- Interaction text contains `https://` → category: **links**
- Text matches personal statement patterns (regex: `/^i (enjoy|built|am|love|wanted)/i`) → category: **personal**

Overrides in `profile.custom.md` can add to or override any category.

The extractor maintains a hardcoded list of company labels to match against: `['PassionGeek', 'G-Next', 'Host Myanmar', 'Arkadium', '2C2P']`. New companies added to the game must be added to this list.

## profile.custom.md Structure

```markdown
# Role & Tone

You ARE Yan Myo Aung. Speak in first person ("I", "me", "my") as if
you're talking directly to the visitor. Be friendly, warm, and genuine.
Keep responses brief — 2 to 4 sentences unless they ask for detail.

# Personal

- I built this pixel art portfolio because I love retro games and wanted
  my portfolio to reflect my personality, not just be a generic form.
- I'm currently studying NCC Education (L5DC, second year) and I hold
  an L4DC Diploma. I'm passionate about learning and growing as a developer.

# Resume

If someone asks about my experience, job history, background, or a CV,
I offer: "I'd be happy to share my resume — you can download it here:
/resume.pdf"

# Off-Topic

If someone asks something completely unrelated to me, my skills, or my
work, I politely redirect: "Hey, I'd rather talk about my work and
projects! Anything you'd like to know about my experience or skills?"
```

## ChatApi Changes

- Constructor accepts `profileCustomContent: string` instead of the old `profileContent`
- The system prompt is assembled at runtime: roleplay rules from `profile.custom.md` + formatted facts from `profile.json`
- `profile.json` is imported at build time (same asset/source webpack rule as .md)
- Guardrails are no longer hardcoded — they come from `profile.custom.md`
- `buildSystemMessage()` merges both sources into the final prompt
- Fallback: if `profile.json` fails to parse, the system prompt uses only `profile.custom.md` content with a warning logged

## ChatWidget Changes

- Add a small 📎 download button next to the "Ask Me Anything" header text
- Button links to `/resume.pdf` (opens in new tab)
- Styled to match existing `chat-close-button` (same font, sizing, color palette)
- Tooltip on hover: "Download Resume (PDF)"

## Build Integration

- `package.json` build script changed to: `"build": "ts-node scripts/extract-profile.ts && webpack --config webpack.config.js --mode production"`
- `package.json` start script changed to: `"start": "ts-node scripts/extract-profile.ts && webpack-dev-server --mode development"`
- `extract-profile.ts` has no dependencies on the game engine — it's a pure data parser that reads `.ts` source files as text
- If the extraction script fails (parse error or empty output), the build fails — webpack never runs
- `profile.json` is gitignored (it's a build artifact, regenerated every build)

## Resume PDF

- User drops their CV as `public/resume.pdf`
- Served as a static file at `/resume.pdf`
- Committed to the repo (not gitignored — it's intentional public content)
- The AI offers it conversationally; the 📎 button provides direct access

## Styling

- Resume download button in chat header: same font family (`Press Start 2P`), color palette (`#8b5a2b` / `#f3e5ab`), and sizing as existing header elements
- Positioned to the right of "Ask Me Anything" text, left of the close X button
- Uses the 📎 (paperclip) emoji as icon

- Chat: "What technologies do you use?" — should list 13+ skills
- Chat: "Tell me about your experience" — should mention companies with descriptions
- Chat: "Can I see your resume?" — should offer download link in first person
- Chat: "What's the weather?" — should politely redirect (off-topic guardrail)
- 📎 button in chat header opens `/resume.pdf` in new tab

## Out of Scope

- Runtime game state reading (contextual chat based on current map)
- Multi-language support
- Conversation persistence across sessions
- Analytics on chat questions
